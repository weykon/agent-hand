## 起動

```bash
agent-hand
```

TUIが開き、左側に**セッションリスト**、下部に**インフォバー**が表示されます。

## 基本キーバインド

| キー | 操作 |
| --- | --- |
| `j` / `k` | セッションリストを上下に移動 |
| `Enter` | 選択したセッションにアタッチ（tmuxにジャンプ） |
| `q` | Agent Handを終了 |
| `/` | セッションをファジー検索 |
| `r` | セッションリストを更新 |
| `g` | グループマネージャーを開く |
| `?` | すべてのキーバインドを表示 |

## セッションの状態

各セッションには色付きのステータスインジケーターが表示されます：

- **Running**（緑）— プロセスがアクティブに実行中
- **Idle**（青）— セッションは存在するが何も実行されていない
- **Exited**（グレー）— プロセスが完了
- **Error**（赤）— プロセスがゼロ以外の終了コードで終了

Agent Handはtmuxペインを検査してこれらの状態を自動的に検出します。

## 優先ジャンプ

セッションは優先度順にソートされます：エラーと実行中のセッションが最初に表示されるため、最も重要なものが常にリストの上部に来ます。

## AIエージェントとの連携

AIコーディングエージェント（Claude Code、Cursorなど）を実行すると、通常複数のターミナルセッションが生成されます。Agent HandはAIツールが作成したセッションを含め、すべてのtmuxセッションを自動的に検出してリスト表示します。

一般的なワークフロー：

1. 1つのターミナルでAIエージェントを起動
2. 別のターミナル（またはtmux分割）でAgent Handを開く
3. セッションがリストに表示されるのを確認
4. 任意のセッションにジャンプして出力を確認
5. ステータスインジケーターでエラーをすばやく発見
