## 概要

フックシステムにより、Agent HandはAIコーディングエージェントからイベントを受信できます。エージェントがセッションを開始したり、プロンプトを送信したり、エラーが発生したりすると、Agent Handはリアルタイムで反応します。

## フックの仕組み

```
AI Agent (例: Claude Code)
    │
    ├─ SessionStart event ──→ hook_event_bridge.sh
    ├─ UserPromptSubmit   ──→ hook_event_bridge.sh
    └─ SessionStop        ──→ hook_event_bridge.sh
                                    │
                                    ▼
                          hook-events.jsonl
                                    │
                                    ▼
                          Agent Hand (EventReceiver)
                                    │
                                    ▼
                          TUIがリアルタイムで更新
```

1. AIエージェントがライフサイクルイベントを発火
2. ブリッジスクリプトがJSONL形式に変換
3. Agent HandのEventReceiverがイベントファイルをポーリング
4. TUIが新しいセッション情報で即座に更新

## サポートされるイベント

| イベント | 説明 |
| --- | --- |
| `SessionStart` | 新しいコーディングセッションが開始された |
| `SessionStop` | セッションが終了した |
| `UserPromptSubmit` | ユーザーがAIにプロンプトを送信した |

## フックのインストール

Agent Handは初回起動時に対応ツールのフックを自動インストールします：

```bash
# フックはツールの設定ディレクトリにインストールされます
# Claude Codeの場合: ~/.claude/hooks/
agent-hand
```

設定ダイアログまたはツールのフック設定を確認することで、フックのインストール状態を確認できます。

## イベントブリッジ

イベントブリッジ（`hook_event_bridge.sh`）は軽量なスクリプトで、以下を行います：

1. AIツールからイベントデータを受信
2. イベントタイプとメタデータを解析
3. イベントファイルにJSONLエントリを書き込み
4. Agent Handが次のポーリングサイクルでそれを取得

ブリッジはステートレスです。永続的な状態やコネクションを保持しません。

## カスタムフック

フックシステムは以下の方法で拡張できます：

1. ブリッジスクリプトに新しいイベントタイプを追加
2. Rustコードベースにカスタムイベントハンドラーを作成
3. `tool-adapters`クレートを使用して新しいAIツールのサポートを追加

## トラブルシューティング

フックが発火しない場合：

1. ブリッジスクリプトが存在し、実行可能であることを確認
2. イベントファイルが書き込まれているか確認: `ls -la ~/.agent-hand/events/`
3. AIツールのフック設定を確認
4. イベントブリッジがイベントディレクトリへの書き込み権限を持っていることを確認
