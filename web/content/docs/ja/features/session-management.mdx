## セッションリスト

メインパネルには検出されたすべてのtmuxセッションが以下の情報とともに表示されます：

- **セッション名** — tmuxセッション識別子
- **ステータス** — 実行中、アイドル、終了、またはエラー（色分け表示）
- **最終アクティビティ** — セッションが最後にアクティブだった時刻
- **グループバッジ** — セッションがグループに属している場合に表示

## ファジー検索

`/` を押すとファジー検索が開きます。名前、グループ、またはラベルでセッションをフィルタリングできます。検索は即座に実行され、すべてのセッションメタデータに対して機能します。

## グループ

グループを使用してセッションを論理的に整理できます。`g` を押してグループマネージャーを開きます。

### グループの作成

1. `g` を押してグループを開く
2. `n` を押して新しいグループを作成
3. グループ名を入力
4. 追加するセッションを選択

### 自動グループ化

Agent Handは命名パターンに基づいてセッションを自動的にグループ化できます。同じAIエージェントやプロジェクトからのセッションは一緒にグループ化されます。

## ラベル

ラベルは、すばやく識別するためにセッションに付けられる軽量なタグです。グループとは異なり、1つのセッションに複数のラベルを付けることができます。

## 優先度ソート

セッションは自動的に優先度順にソートされます：

1. **Error** — ゼロ以外の終了コードのセッション（最上位に表示）
2. **Running** — アクティブに実行中のプロセス
3. **Idle** — 入力待ち
4. **Exited** — 完了したプロセス（最下位に表示）

これにより、最も重要なセッションが常に最初に表示されます。

## セッション操作

| キー | 操作 |
| --- | --- |
| `Enter` | セッションにアタッチ |
| `d` | セッションからデタッチ |
| `x` | セッションを終了（確認あり） |
| `g` | グループを管理 |
| `/` | ファジー検索 |
| `r` | リストを更新 |
| `s` | ソートオプション |

## マルチツール対応

複数のAIエージェントを同時に使用している場合、Agent Handはすべてのセッションを1つのビューに統合します。各エージェントのセッションは、命名パターンとオプションのグループ割り当てによって区別できます。
